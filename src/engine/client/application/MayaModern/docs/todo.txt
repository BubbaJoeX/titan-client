# SwgMayaEditor (MayaModern) - Maya 2026
# Full feature parity with MayaExporter (32-bit legacy) - migrated to 64-bit Maya 2026 with an Importer option.

## Build Requirements
- Maya 2026 (64-bit) - https://www.autodesk.com/developer-network/platform-technologies/maya
- Maya 2026 Devkit - **Separate download** from Autodesk Feedback Forum (Downloads section). Extract include/ and lib/ into Maya dir or set MAYA_LOCATION to devkit root.
- Visual Studio 2022 (17.8.3+)
- CMake 3.13+
- Configure: cmake -B build -A x64 (Windows 64-bit required for Maya 2026)
- FindMaya checks: MAYA_LOCATION, C:\Program Files\Autodesk\Maya2026, D:\Program Files\Autodesk\Maya2026

## Supported Use Cases (from original todo.txt)
- Modifying hard points/collision on existing objects
- Adding rooms to a structure that already exists
- Changing existing animations (or animation tables?)
- Creating a new structure/ship pob
- Creating a new static object
- Creating a new vehicle
- Creating a new species/skeletal body and animations
- Creating a new wearable/object with occlusion/form modification

### Pipeline Workflow
1. **Import** - Load SWG assets (mesh, skeleton, animation, materials)
2. **Edit** - Modify in Maya (hardpoints, blend shapes, geometry, materials)
3. **Export** - Write back to SWG IFF format

Current state: Import is largely complete; static mesh + POB + skeleton + shader export paths exist. Skeletal `.mgn`, `.ans`, `.sat` **binary export** remains legacy MayaExporter scope unless ported.
